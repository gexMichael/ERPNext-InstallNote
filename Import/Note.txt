/*
  匯入資料時，要注意中文欄位抬頭
  因為我採用了余老師的漢化方案，而且轉為正體版
  這會和用 VM 版的中文不同，會造成匯入時欄位抬頭無法對照
  
  導致無法匯入資料
  解法就是先確認中文的翻譯版本內容，盡量不要隨意的異動
  如果有變動，匯入檔案的欄位抬頭記得修改即可
  
  2024/6/6 Michael

1.所以，前期要先確認中文翻譯都確認正確，這會影響所有的匯入及查詢框。
2.物料組匯入時如果有設定 Is Group, 只能先將 group 先匯入, 再匯入尾階群組
  也就是要分批匯入，一次匯入會報錯，因為部分 group 還沒有匯入
  
3.匯入的次序如下
  a.Account 會計科目
    >> 會計科目修正-01-補寫台灣會計科目.csv
  b.Users (要注意不要被 role 卡住)
    >> users.csv
  c.employee (非必要)
    >> employee.csv
  d.供應商
    >> 先手動建 供應商類型
	   原廠直供
	   代理商
	   周邊雜項
    >> 供應商-20240601.csv
  e.客戶 
    >> 要先手動建立客戶群組
	   經銷商
	   代理商
	   企業客戶
	>> 還要手動建立 默認價格表
	   經銷價
	   代理商價
	   零售價
	>> 客戶-20240527.csv
  
  f.物料組
    >> 要注意 【All Item Groups】or【所有物料組】
    >> 物料組-20240606-root-1.csv
	>> 物料組-20240606-root-2.csv
	>> 物料組-20240606-left.csv
	
  g.產品(物料)
    >> 基本單位要新增 PCS
	>> 倉庫要調整，因為新增物流主倉及【非存貨倉】
	   MI-小米倉  >>> 觀音物流倉(北)
	   DJI-大疆倉 >>> 烏日物流倉(中)
    >> 產品-Items-all_v3.csv
	
  h.物料價格
    >> 物料價格.csv
